# 🤖 GenBlog — AI-Powered Blog Generation Platform

> Transform any YouTube video, audio file, PDF document, or text topic into a fully polished, SEO-optimized blog post in seconds — powered by Groq LLaMA, AssemblyAI, and Pollinations.ai FLUX.

---

## ✨ Features

| Feature | Description |
|---|---|
| 🎥 **YouTube → Blog** | Paste a YouTube URL → audio is downloaded via yt-dlp → transcribed by AssemblyAI → Groq writes the blog |
| 🎙️ **Audio → Blog** | Upload any audio file → Gemini transcribes → Groq writes the blog |
| 📄 **PDF → Blog (RAG)** | Upload a PDF → stored in ChromaDB → used as context for Groq blog generation |
| ✍️ **Text → Blog** | Enter a topic or text → Groq generates a full blog post with SEO metadata |
| 🖼️ **AI Thumbnail** | Blog images generated by Pollinations.ai (FLUX) |
| 📊 **Content Scoring** | Auto-computes SEO Score, Readability Score, and Engagement Score |
| 🔁 **Blog Versioning** | Every generation and regeneration is saved as a versioned snapshot |
| 👤 **User Profiles** | Auth system with custom user model, profile image, public blog listing |
| 💬 **Explore & Like** | Discover and like community blogs |
| 📬 **Contact Form** | Built-in contact messages saved to DB |

---

## 🏗️ Architecture & Flow

```
YouTube URL
    │
    ├─► [yt-dlp]             Download best audio (m4a/webm) — no FFmpeg needed
    │
    ├─► [AssemblyAI]         Transcribe audio using universal-3-pro speech model
    │
    └─► [Groq LLaMA-3.3-70B] Generate blog + SEO metadata (single API call)
            │
            └─► [Pollinations.ai FLUX]  Generate blog thumbnail image
                        │
                        └─► [Django + SQLite]  Save blog post, version, thumbnail
```

```
Text / Topic
    │
    ├─► [DuckDuckGo Search]  (optional) Web research context
    ├─► [ChromaDB RAG]       (optional) PDF document context
    │
    └─► [Groq LLaMA-3.3-70B] Generate blog + SEO metadata (single API call)
```

## ⚙️ Setup & Installation

### 1. Clone the repository
```bash
git clone https://github.com/atul120212/GenBlog.git
cd GenBlog
```

### 2. Create virtual environment
```bash
python -m venv .venv
.venv\Scripts\activate     # Windows
# source .venv/bin/activate  # macOS/Linux
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
pip install groq assemblyai yt-dlp
```

### 4. Configure environment variables

Create a `.env` file in the project root:
```env
GEMINI_API_KEY=your_gemini_api_key_here
GROQ_API_KEY=your_groq_api_key_here
ASSEMBLYAI_API_KEY=your_assemblyai_api_key_here
CONTACT_EMAIL=your_email@example.com
```

**Where to get free API keys:**
| Service | Free Tier | Link |
|---|---|---|
| Groq | 14,400 req/day | [console.groq.com](https://console.groq.com) |
| AssemblyAI | 100 hours/month | [assemblyai.com](https://www.assemblyai.com) |
| Pollinations.ai | 100% Free / No Key | [pollinations.ai](https://pollinations.ai) |
| Gemini | 1,500 req/day | [aistudio.google.com/apikey](https://aistudio.google.com/apikey) |

### 5. Run migrations
```bash
python manage.py migrate
```

### 6. Start the development server
```bash
python manage.py runserver
```

Visit **http://127.0.0.1:8000**

## 🧠 AI Agent Pipeline (agents.py)

The entire pipeline is designed around **one Groq API call per blog** to maximize free-tier efficiency:

```
ResearchAgent (zero API calls)
    ├── DuckDuckGo web search (optional)
    └── ChromaDB RAG context (optional)

BlogGenerationWorkflow (ONE Groq call)
    └── Returns: blog content (Markdown) + SEO metadata (JSON)

generate_blog_image (Pollinations.ai call, free/no key, non-blocking)
    └── Returns: PNG bytes or None (failure is silent)
```

**Content Scoring** (zero API calls, local computation):
- **SEO Score** — checks word count, meta tags, headings, keyword density
- **Readability Score** — Flesch Reading Ease via `textstat`
- **Engagement Score** — heuristic: questions, lists, bold text, exclamations
